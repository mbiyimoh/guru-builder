# Guru Builder System - UX Design & Workflow

## 1. User Workflow & User Stories

### Core Jobs to Be Done

**Primary JTBD**: "Help me build a comprehensive, well-structured AI teaching assistant for [game/domain] by iteratively researching and incorporating expert knowledge."

### User Personas

1. **Guru Creator** - Someone with domain expertise wanting to create an AI teaching assistant
2. **Content Curator** - Someone managing and refining an existing guru's knowledge base

### User Stories & Workflow

#### Phase 1: Guru Initialization
```
AS A guru creator
I WANT TO set up a new guru project with initial teaching philosophy and foundational knowledge
SO THAT I have a starting point for iterative improvement

Acceptance Criteria:
- Create new guru project with name/description
- Upload or write initial context layer (Layer 0: Teaching Philosophy)
- Optionally add Layer 1 (Foundational Principles)
- Optionally upload initial knowledge files
- Save and view current guru configuration
```

#### Phase 2: Research Run Configuration
```
AS A guru creator
I WANT TO define a research task for the system to execute
SO THAT the guru can learn from external sources and recommend improvements

Acceptance Criteria:
- Specify research instructions (URL, topic, expert name, etc.)
- Choose research depth (quick scan vs deep analysis)
- Define scope (which layers/files should be considered for updates)
- Submit research run and track progress
```

#### Phase 3: Research Execution & Analysis
```
AS A system
I NEED TO consume external content and analyze it against existing corpus
SO THAT I can generate actionable recommendations

Process:
1. Deep research on specified sources using foundation model capabilities
2. Digest and synthesize information
3. Compare to existing context layers and knowledge files
4. Generate structured recommendations:
   - Top 3-5 priority recommendations with justifications
   - Additional recommendations bucketed by impact/conviction (high/medium/low)
   - Each recommendation specifies: action type (add/edit/delete), target (layer/file), proposed changes
```

#### Phase 4: Review & Approve Recommendations
```
AS A guru creator
I WANT TO review structured recommendations from the research run
SO THAT I can approve valuable changes and reject inappropriate ones

Acceptance Criteria:
- View research summary (what was learned)
- See top priority recommendations first
- For each recommendation:
  - View justification and confidence level
  - Preview before/after comparison
  - Approve, reject, or edit the recommendation
  - Add notes for future reference
- Bulk actions (approve all high-confidence, reject all low-confidence)
- Save decisions and proceed to execution
```

#### Phase 5: Apply Changes
```
AS A system
I NEED TO execute approved recommendations
SO THAT the guru's corpus is updated with new knowledge

Process:
1. Execute approved changes in order
2. Create/edit/delete context layers
3. Create/edit/delete knowledge files
4. Generate summary report of changes made
5. Version control (store snapshot before changes)
```

#### Phase 6: Iterate
```
AS A guru creator
I WANT TO run additional research cycles
SO THAT I can continuously improve the guru's knowledge

Acceptance Criteria:
- View change history
- Rollback to previous versions if needed
- Start new research run with different sources
- Export final guru configuration
```

## Key UX Design Principles

This UX design emphasizes:

1. **Progressive disclosure** - Start simple (dashboard) â†’ drill into details
2. **Transparency** - Always show what's in the corpus, what changed, why
3. **Control** - User approval required for all changes
4. **Structured data** - Recommendations are actionable objects, not just text
5. **Iteration** - Easy to run multiple research cycles
6. **Safety** - Automatic backups before changes
7. **Context** - Clear connections between layers, files, and references
8. **Confidence signaling** - Visual indicators of recommendation strength

The workflow allows creators to systematically build comprehensive AI tutors through iterative research and curation, while maintaining full control over what knowledge gets incorporated.
